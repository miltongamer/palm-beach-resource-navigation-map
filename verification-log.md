# Verification Log

Use this file to track updates, source checks, corrections, and changes to the resource database.

## Verification standards

Each resource should be checked for:

- Organization name
- Phone number
- Website
- Address
- Hours
- Services offered
- Eligibility requirements
- Referral process
- Cost/payment options
- Service area
- Transportation availability
- Crisis/emergency limitations
- Last verified date

## Verification status definitions

- **Verified** — Confirmed directly through official website, phone call, email, or recent official document.
- **Partially verified** — Some key details confirmed, but important details still need follow-up.
- **Needs verification** — Resource has not been checked recently or source reliability is uncertain.
- **Outdated** — Information appears old, contradicted, or no longer reliable.
- **Removed** — Resource no longer appears active or no longer fits the project.

## Confidence level definitions

- **High** — Official source or direct confirmation within the last 90 days.
- **Medium** — Official source found, but not recently confirmed directly.
- **Low** — Third-party source, unclear date, or information needs confirmation.
